The Vue, React, Svelte, and Solid.js packages each ship a Vite plugin at `@unhead/{framework}/vite`. Solid.js uses the package name `@unhead/solid-js`.

## Setup

Install your framework's Unhead package, then add the plugin to your Vite config:

::FrameworkCode

#vue
```ts [vite.config.ts]
import vue from '@vitejs/plugin-vue'
import { Unhead } from '@unhead/vue/vite'

export default defineConfig({
  plugins: [vue(), Unhead()],
})
```

#react
```ts [vite.config.ts]
import react from '@vitejs/plugin-react'
import { Unhead } from '@unhead/react/vite'

export default defineConfig({
  plugins: [react(), Unhead()],
})
```

#svelte
```ts [vite.config.ts]
import { svelte } from '@sveltejs/vite-plugin-svelte'
import { Unhead } from '@unhead/svelte/vite'

export default defineConfig({
  plugins: [Unhead(), svelte()],
})
```

#solid
```ts [vite.config.ts]
import solid from 'vite-plugin-solid'
import { Unhead } from '@unhead/solid-js/vite'

export default defineConfig({
  // Must come before solid() to see JSX before compilation
  plugins: [Unhead(), solid()],
})
```

#nuxt
```ts
// No configuration needed
```

::

## Included transforms

The plugin combines several build-time optimizations:

- **[Tree-shaking](/docs/head/guides/build-plugins/tree-shaking)**: Removes deprecated server composables and `useSchemaOrg` calls from client bundles
- **[useSeoMeta transform](/docs/head/guides/build-plugins/seo-meta-transform)**: Converts `useSeoMeta()` calls into raw `useHead()` calls at build time
- **[Minify transform](/docs/head/guides/build-plugins/minify-transform)**: Pre-minify static inline script/style content at build time
- **Static precompilation (experimental)**: Compile sealed client and server entries into render-ready plans
- **Dev validation**: Auto-injects `ValidatePlugin` in dev so head tag warnings surface in the browser console (enabled by default)
- **[DevTools](/docs/head/guides/build-plugins/devtools)**: Registers Unhead's Vite DevTools integration by default; the panel activates when Vite DevTools is enabled

## Options

```ts
Unhead({
  // Disable server composable tree-shaking
  treeshake: false,
  // Disable useSeoMeta → useHead transform
  transformSeoMeta: false,
  // Pre-minify inline script/style content
  minify: { js: createJSMinifier(), css: createCSSMinifier() },
  // Compile sealed static server entries at build time
  experimental: { precompile: true },
  // File filter (shared across all transforms)
  filter: { exclude: [/some-file/] },
})
```

| Option | Type | Default | Description |
| -------- | ------ | --------- | ------------- |
| `treeshake` | `object \| false` | enabled | [Tree-shake server composables](/docs/head/guides/build-plugins/tree-shaking) from client bundles |
| `transformSeoMeta` | `object \| false` | enabled | [Transform `useSeoMeta()`](/docs/head/guides/build-plugins/seo-meta-transform) to `useHead()` |
| `minify` | `object \| false` | disabled | [Pre-minify static inline script/style](/docs/head/guides/build-plugins/minify-transform) |
| `experimental.precompile` | `boolean \| object` | disabled | Compile sealed `unhead/precompiled/client` and `/server` calls into render-ready plans |
| `validate` | `boolean` | enabled | Auto-inject `ValidatePlugin` in dev for head tag warnings |
| `devtools` | `object \| false` | included | [Vite DevTools integration](/docs/head/guides/build-plugins/devtools) (dev only) |
| `filter` | `object` | Unset | Shared include/exclude file filter |
| `sourcemap` | `boolean` | Unset | Enable sourcemap generation for transforms |

The three transforms accept `false` or an options object. `devtools` follows the same pattern, while `validate` and `sourcemap` are booleans and `filter` is an object.

## Static Precompilation

Static precompilation provides separate sealed client and server targets. It moves normalization, identity, sanitization, attributes, and tag content into the build. Server plans contain final HTML; client plans contain DOM-ready records with disposal support.

```ts [vite.config.ts]
Unhead({
  experimental: {
    precompile: true,
  },
})
```

```ts
import { createHead, renderSSRHead, useHead } from 'unhead/precompiled/server'

const head = createHead()
useHead({
  title: 'Product',
  meta: [{ name: 'description', content: 'Static product page' }],
}, { head })

const payload = renderSSRHead(head)
```

```ts
import { createHead, useHead } from 'unhead/precompiled/client'

const head = createHead()
const entry = useHead({ title: 'Product' }, { head })

// Client entries can be removed during navigation.
entry.dispose()
```

The mode is compile-or-error. Unsupported syntax fails the build with its file, line, and reason rather than preserving a large fallback runtime.

| Supported | Rejected or excluded |
|---|---|
| JSON-compatible `useHead()` literals | Variables, functions, refs, getters, spreads, and computed keys |
| Static `useSeoMeta()` literals | `patch()` and runtime input replacement |
| Package-owned pre-normalized defaults | Raw `init`, plugins, hooks, custom `tagWeight`, and custom `propResolvers` |
| Server `disableDefaults`; compact rendering without line breaks | Other create/render options and entry options other than `{ head }` |
| Static tag priority and valid position | `titleTemplate`, invalid tag positions, `templateParams`, `processTemplateParams`, and custom duplicate strategies |
| Scalar arrayable SEO metadata | Repeated arrayable identities, explicit tag keys, and class/style attributes |
| Plain HTML/body attributes | Framework server entries and streaming replay |

Use `unhead/server` or a framework server entry when you need any excluded capability.

The sealed renderer always omits formatting line breaks. This removes a per-fragment branch and produces the smallest response; line-break formatting is intentionally not dynamic in this mode.

Repeated arrayable identities are excluded. For example, one call cannot emit two `og:image` tags or two `theme-color` tags. Use the normal target when multiple values must survive from one entry.

Server calls must come from `unhead/precompiled/server`; client calls must come from `unhead/precompiled/client`. A target mismatch fails the build instead of leaking the wrong runtime. Imports from ordinary Unhead and framework entries stay unchanged.

The client target keeps entry disposal so route unmounts can remove a static plan. It adopts matching SSR elements, reconciles changed winners, and never replays an unchanged script. It excludes entry patches, event handlers, hooks, plugins, custom documents, and runtime normalization.

Vite detects client and SSR builds automatically. For plain Rollup, or another bundler that cannot expose its target, set `consumer: 'client'` or `consumer: 'server'` explicitly.

## Other Bundlers

The Vue, React, Svelte, and Solid.js packages expose a unified `/bundler` entry with per-bundler methods:

```ts
import { Unhead } from '@unhead/vue/bundler'

export default {
  plugins: Unhead().webpack(),
}
```

The bundler entry also exposes `.rspack()` and `.rollup()`. Validation and DevTools injection are Vite-only. Rollup cannot infer whether it is building for the client or server, so target-dependent transforms remain conservative; use `.vite()` or `.webpack()` when you need target-aware optimization.
