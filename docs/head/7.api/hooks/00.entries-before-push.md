The `entries:beforePush` hook runs synchronously before Unhead creates an entry. Set `ctx.shouldPush` to `false` to discard the input.

A suppressed push returns an inert `ActiveHeadEntry`. Calling `patch()` or `dispose()` on that entry has no effect, including after the hook is removed.

## Hook Signature

```ts
export interface Hook {
  'entries:beforePush': (ctx: {
    readonly input: unknown
    readonly options: Readonly<HeadEntryOptions>
    shouldPush: boolean
  }) => void
}
```

### Parameters

| Name | Type | Description |
| ------ | ------ | ------------- |
| `ctx.input` | `unknown` | Input passed to `head.push()` |
| `ctx.options` | `Readonly<HeadEntryOptions>` | Entry options passed to `head.push()` |
| `ctx.shouldPush` | `boolean` | Whether Unhead should create the entry, defaults to `true` |

## Usage Example

An island renderer can ignore head writes made while application plugins run, then accept entries from the island component render:

```ts
const removeHook = head.hooks.hook('entries:beforePush', (ctx) => {
  ctx.shouldPush = false
})

await installApplicationPlugins()
removeHook()

await renderIslandComponent()
```

The hook must be synchronous. Use it to accept or reject a push, not to resolve input values.
