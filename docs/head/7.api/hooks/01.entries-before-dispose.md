The `entries:beforeDispose` hook runs synchronously before a client entry is removed. Call `ctx.defer()` with a promise to keep the entry active until the promise settles.

Unhead waits for every promise registered by the hook. Rejected promises still release the entry, which ensures failed navigations do not leak head state. Repeated `dispose()` calls while disposal is deferred have no effect.

The server adapter does not emit this hook because server entries do not drive DOM lifecycle updates.

## Hook Signature

```ts
export interface Hook {
  'entries:beforeDispose': (ctx: {
    readonly entry: HeadEntry<unknown>
    defer: (promise: PromiseLike<unknown>) => void
  }) => void
}
```

### Parameters

| Name | Type | Description |
| ------ | ------ | ------------- |
| `ctx.entry` | `HeadEntry<unknown>` | Entry that is about to be disposed |
| `ctx.defer` | `(promise: PromiseLike<unknown>) => void` | Keeps the entry active until the promise settles |

## Usage Example

Keep the outgoing page metadata active until a route transition completes:

```ts
let transitionPromise: Promise<unknown> | undefined

head.hooks.hook('entries:beforeDispose', (ctx) => {
  if (transitionPromise)
    ctx.defer(transitionPromise)
})
```

Call `defer()` during the hook. Promises registered after an `await` are too late to delay disposal.
