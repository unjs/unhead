title: 'Wrapping Composables'
description: 'Learn how to create custom head composables by wrapping useHead with your own defaults and context management.'
---

Store the head instance in your application context, then wrap the core composables so callers do not have to pass it each time.

## Why Wrap the Composables

When you create an Unhead instance, the framework-agnostic package does not register it in a global context. You must pass the instance to each core composable.

Framework adapters manage this context for you. In a framework-free application, you can wrap the core composables around your own application context.

The installation guide shows how to attach the head instance to the global window object:

```ts
import { useHead } from 'unhead'

// ❌ Not recommended
useHead(window.__UNHEAD__, {
  title: 'My Page',
})
```

A fixed window property can collide when several copies of an application run on the same page. It also exposes the instance outside the application's own context.

## Create an Application Context

Use the context mechanism provided by your application or framework. This example uses [unctx](https://github.com/unjs/unctx):

```ts [my-app-context.ts]
import type { Unhead } from 'unhead/types'
import { createContext } from 'unctx'

export interface MyApp {
  head: Unhead
}

const appCtx = createContext<MyApp>()

export const useMyApp = appCtx.use
export const callWithMyApp = appCtx.call
```

```ts [main.ts]
import { createHead } from 'unhead/client'
import { callWithMyApp } from './my-app-context'
import { startApp } from './start-app'

const app = { head: createHead() }

callWithMyApp(app, () => {
  startApp()
})
```

`unctx` makes the value available while the callback runs. Capture the app context before an `await`, or configure its async-context support if your application needs context to cross asynchronous boundaries.

## Wrap `useHead()`

Wrap each composable to read the head instance from that context:

```ts [head.ts]
import type { HeadEntryOptions, ResolvableHead } from 'unhead/types'
import { useHead as baseUseHead } from 'unhead'
import { useMyApp } from './my-app-context'

export function useHead(input: ResolvableHead, options?: HeadEntryOptions) {
  const { head } = useMyApp()
  return baseUseHead(head, input, options)
}
```

```ts
import { useHead } from './head'

// ✅ Recommended
useHead({
  title: 'My Page',
})
```
