## Introduction

[SvelteKit](https://kit.svelte.dev/) provides its own SSR infrastructure that differs from the basic Vite SSR template. This guide covers how to wire up Unhead using SvelteKit's `handle` hook and Svelte context API.

**Quick Start:** Install `@unhead/svelte`, create the head in `hooks.server.ts`, pass it through `event.locals`, provide it via context in `+layout.svelte`, and call `useHead()` in your pages.

## Setup

### 1. Install the package

:ModuleInstall{name="@unhead/svelte@next"}

### 2. Update app.d.ts

Extend SvelteKit's `Locals` interface so the head instance can flow through the request:

```ts [src/app.d.ts]
import type { Unhead } from '@unhead/svelte/server'

declare global {
  namespace App {
    interface Locals {
      unhead: Unhead
    }
  }
}

export {}
```

### 3. Create the head in hooks.server.ts

Use the `handle` hook to create a head instance per request and inject the rendered tags into the HTML response:

```ts [src/hooks.server.ts]
import { createHead, transformHtmlTemplate } from '@unhead/svelte/server'
import type { Handle } from '@sveltejs/kit'

export const handle: Handle = async ({ event, resolve }) => {
  const unhead = createHead()
  event.locals.unhead = unhead

  const response = await resolve(event, {
    transformPageChunk: async ({ html }) => {
      return transformHtmlTemplate(unhead, html)
    },
  })

  return response
}
```

`transformHtmlTemplate()` injects the rendered head tags into placeholders in SvelteKit's HTML template — including `<title>`, `<meta>`, and any other tags set via `useHead()`.

### 4. Provide the head in +layout.server.ts

Expose the head instance to the layout via load data:

::note
SvelteKit serialises `load` return values as JSON, so we can't pass the `unhead` object directly. Instead, we pass the rendered SSR payload and reconstruct on the client.
::

The cleanest approach is to pass the head through Svelte's context API rather than load data. Do this in the root layout:

```ts [src/routes/+layout.server.ts]
import type { LayoutServerLoad } from './$types'

export const load: LayoutServerLoad = async ({ locals }) => {
  return {
    // expose any initial head data needed by the client here
  }
}
```

### 5. Provide the head instance in +layout.svelte

On the server, retrieve the head from `locals` via `$page.data` or a server load function. On the client, create a fresh client-side head and provide both via Svelte context:

```svelte [src/routes/+layout.svelte]
<script lang="ts">
  import { setContext, onMount } from 'svelte'
  import { page } from '$app/stores'
  import { browser } from '$app/environment'
  import { UnheadContextKey } from '@unhead/svelte'
  import { createHead } from '@unhead/svelte/client'

  // Create the client head once (no-op on server — SvelteKit renders
  // this file on both sides, so guard with browser check)
  const unhead = browser ? createHead() : null

  if (unhead) {
    setContext(UnheadContextKey, unhead)
  }
</script>

<slot />
```

::note
On the server, Unhead is wired up via `hooks.server.ts`. The layout only needs to provide the client-side instance so that `useHead()` calls in `+page.svelte` work after hydration.
::

### 6. Use useHead() in your pages

```svelte [src/routes/+page.svelte]
<script lang="ts">
  import { useHead, useSeoMeta } from '@unhead/svelte'

  // Static tags
  useHead({
    title: 'My SvelteKit App',
    meta: [
      { name: 'description', content: 'Built with SvelteKit and Unhead' }
    ]
  })
</script>

<h1>Home</h1>
```

For reactive tags driven by page data:

```svelte [src/routes/blog/[slug]/+page.svelte]
<script lang="ts">
  import { useSeoMeta } from '@unhead/svelte'
  import type { PageData } from './$types'

  let { data }: { data: PageData } = $props()

  const entry = useSeoMeta()

  $effect(() => {
    entry.patch({
      title: data.post.title,
      description: data.post.excerpt,
      ogImage: data.post.coverImage,
    })
  })
</script>
```

### 7. Add default tags (optional)

Set site-wide defaults in `hooks.server.ts` using the `init` option:

```ts [src/hooks.server.ts]
import { createHead, transformHtmlTemplate } from '@unhead/svelte/server'
import type { Handle } from '@sveltejs/kit'

export const handle: Handle = async ({ event, resolve }) => {
  const unhead = createHead({
    init: [
      {
        htmlAttrs: { lang: 'en' },
        title: 'My SvelteKit App',
        titleTemplate: '%s | My Site',
        meta: [
          { name: 'description', content: 'Default site description' }
        ],
      },
    ],
  })
  event.locals.unhead = unhead

  const response = await resolve(event, {
    transformPageChunk: async ({ html }) => {
      return transformHtmlTemplate(unhead, html)
    },
  })

  return response
}
```

## How the pieces fit together

| File | Role |
|---|---|
| `hooks.server.ts` | Creates a per-request head instance, transforms HTML |
| `+layout.svelte` | Provides the client-side head via Svelte context |
| `+page.svelte` / any component | Calls `useHead()` / `useSeoMeta()` |

On the **server**, `hooks.server.ts` creates the head, Svelte renders the component tree (collecting `useHead()` calls), and `transformHtmlTemplate()` injects the tags before the response is sent.

On the **client**, the layout creates a new head instance and provides it via context. SvelteKit hydrates the page and subsequent navigation triggers reactive updates through `$effect()`.
