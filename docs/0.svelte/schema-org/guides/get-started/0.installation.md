## Setup

### 1. Install the Package

Install `@unhead/schema-org` in your project:

::code-group

```bash [yarn]
yarn add @unhead/schema-org
```

```bash [npm]
npm install @unhead/schema-org
```

```bash [pnpm]
pnpm add @unhead/schema-org
```

::

### 2. Configure Schema.org Params

At a minimum, provide a [`host`](/docs/schema-org/guides/core-concepts/params):

```svelte
<script lang="ts">
  import { useHead } from '@unhead/svelte'

  useHead({
    templateParams: {
      schemaOrg: {
        host: 'https://example.com',
      }
    }
  })
</script>
```

See [Schema.org Params](/docs/schema-org/guides/core-concepts/params) for all available options.

### 3. Add Site Schema.org

Import from the Svelte adapter so `useSchemaOrg()` gets the head instance from Svelte context and registers component cleanup:

```svelte
<script lang="ts">
  import { defineWebPage, defineWebSite, useSchemaOrg } from '@unhead/schema-org/svelte'

  useSchemaOrg([
    defineWebSite({ name: 'Example Site' }),
    defineWebPage(),
  ])
</script>
```

## Recommended: Vite Plugin

The [Vite plugin](/docs/head/guides/build-plugins/overview) can tree-shake server-only code and transform head composables at build time:

```ts [vite.config.ts]
import { svelte } from '@sveltejs/vite-plugin-svelte'
import { Unhead } from '@unhead/svelte/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [svelte(), Unhead()],
})
```

## Optional: Auto-Imports

The Svelte adapter exports an auto-import preset:

```ts [vite.config.ts]
import { schemaOrgAutoImports } from '@unhead/schema-org/svelte'
import AutoImport from 'unplugin-auto-import/vite'

export default defineConfig({
  plugins: [
    AutoImport({
      imports: schemaOrgAutoImports,
    }),
  ],
})
```
